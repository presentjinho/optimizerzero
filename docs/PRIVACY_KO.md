# OptimizerZero 개인정보/로컬 처리 메모

OptimizerZero Web Lite는 브라우저 안에서 파일을 처리하는 방식입니다.

## 기본 원칙

- 사용자가 고른 파일은 브라우저 안에서 처리됩니다.
- OptimizerZero 서버로 파일을 업로드하지 않습니다.
- 분석 스크립트, 추적 픽셀, 원격 로그 기능을 넣지 않았습니다.
- JSZip은 `web/vendor/`에 포함되어 배포 후 외부 CDN 호출 없이 동작합니다.
- 리포트 파일은 사용자가 직접 저장 버튼을 눌렀을 때만 만들어집니다.

## 브라우저 저장

- 서비스워커가 앱 파일을 캐시해서 다음 접속과 오프라인 재실행을 돕습니다.
- 최적화 결과물은 브라우저 다운로드로 생성됩니다.
- 선택한 원본 파일을 앱 저장소에 의도적으로 보관하지 않습니다.

## 주의

- 매우 큰 파일은 브라우저 메모리 제한 때문에 실패할 수 있습니다.
- Cloudflare Access 같은 로그인 보호를 붙이면 해당 서비스의 접속 로그는 남을 수 있습니다.
- 나중에 분석 도구나 서버 업로드 기능을 추가하면 이 문서를 함께 수정해야 합니다.
